# Chaff — Demo Run-of-Show

A tight 2-minute demo built to win on both axes: instant peer "wow" + real
technical depth for judges.

## Setup checklist (before you present)

- [ ] Backend running and serving the built frontend → open `http://localhost:8000`
      (or run the Vite dev server). Globe should be spinning with attacks.
- [ ] Internet optional: the earth texture is bundled locally (`/earth-night.jpg`).
- [ ] Know the vault password (default in `backend/app/vault.py`,
      or whatever `CHAFF_VAULT_PASSWORD` you set).
- [ ] Browser zoom set so the globe + sidebar both fit on screen.
- [ ] Backup: a screen recording of a successful run, in case of A/V issues.

## The 2 minutes

1. **Hook (15s).** Gesture at the globe. *"Every arc you see is a brute-force
   attempt hitting our honeypot. Attackers do this to any exposed server within
   minutes. Watch the counter climb."* Point at **Attacks captured**.

2. **The twist (20s).** *"Most defenses just block attackers. We do the
   opposite — we let them in, and feed them lies."* Open the **Honey Vault**
   panel; note the scheme line (Honey Encryption, 120k iterations).

3. **The reveal (35s).**
   - Type a wrong password (e.g. `letmein`) → **Crack** → a real-looking credential.
   - Type the **real** password → the real credential appears.
   - *"Here's the point: the interface gave no hint which was real — because the
     server itself can't tell. Every wrong key decrypts to a different, valid,
     believable credential."*

4. **The money shot (25s).** Click **Run dictionary attack**. The screen floods
   with believable credentials; the **Decoys served** counter rockets. *"They just
   'stole' thousands of credentials. All fake. And they can never tell which — if any
   — is real. Brute force is now worthless."*

5. **Hand it to the room (20s).** Open **Spot the Real Secret**. *"One of these six
   is the real secret. Anyone — point at the real one."* Let a peer pick → reveal.
   *"You couldn't. Nobody can. That's Honey Encryption."*

6. **Close (5s).** *"Chaff: we don't just watch attackers — we drown them in
   believable lies. Live attack intelligence on top, deception crypto underneath."*

## Why it wins (say this if asked)

- **Peer vote:** a live world map of real attacks + a game you can play in 5
  seconds. Instant and interactive.
- **Judge vote:** real cryptography (Honey Encryption — Juels & Ristenpart,
  2014) + active cyber-deception + a real honeypot data path. Not an LLM wrapper.

## Anticipated questions

- **"Is this just calling an API?"** No. The core is a Distribution-Transforming
  Encoder + PBKDF2 honey-encryption scheme we implemented; 19 unit tests prove
  every key yields a valid credential. See `backend/app/honey/`.
- **"Are the attacks real?"** Yes — and we prove it on screen. The top-bar badge
  reads **LIVE · real honeypot** when the feed is a real Cowrie log (vs
  **SIMULATED**), and `GET /api/source` returns `{"mode":"cowrie"}`. Point
  `CHAFF_COWRIE_LOG` at a live Cowrie honeypot and the globe shows genuine
  attackers; the built-in simulator is only the offline fallback.
- **"How does the legitimate owner know they got the real credential?"** They know
  their own password. Honey Encryption protects against *brute-force attackers*,
  who have no such oracle.
- **"Doesn't a wrong password ever produce garbage?"** Never — by construction
  the DTE maps every seed to a valid credential, so wrong keys yield valid decoys.
- **"Is the credential data real?"** Entirely synthetic, generated by the DTE.

## If something breaks

- Globe blank → check the texture loads; the simulator still streams data
  regardless. Refresh.
- Backend down → restart uvicorn; the demo is stateless.
- No network → everything still works; the earth texture and flag font are
  bundled locally, and the simulator stands in if the honeypot is unreachable.
